Documentation

Everything you need to run and use Waypoint. For the code, issues, and releases, see GitHub — deployment and configuration guides also live in the repo as markdown under docs/ (including a full Kubernetes example).

Quick start

The repo ships a compose file with the app and Postgres:

git clone https://github.com/ctrl-research/waypoint
cd waypoint
cp .env.example .env      # set POSTGRES_PASSWORD at minimum
docker compose up -d

The app listens on :8080. Prebuilt multi-arch images (amd64/arm64) are on GHCR if you'd rather point your own deployment at them:

docker pull ghcr.io/ctrl-research/waypoint:latest   # or a specific X.Y.Z

Database migrations run automatically at startup. The first account to sign in becomes admin, and a guided tour with demo data offers itself on first login (replayable later from Settings).

For development without Docker, see the repo README — make dev runs the Go server and Vite with hot reload.

Configuration

Everything is configured with WAYPOINT_* environment variables:

VariableDefaultWhat it does
WAYPOINT_DATABASE_URL—Required. Postgres connection string.
WAYPOINT_ADDR:8080Listen address.
WAYPOINT_BASE_URLhttp://localhost:8080Public URL of the instance. OAuth redirect URIs derive from it; cookies are Secure when it's https.
WAYPOINT_GOOGLE_CLIENT_ID / WAYPOINT_GOOGLE_CLIENT_SECRET—Enable Sign in with Google (both together). See Google sign-in.
WAYPOINT_LOCAL_AUTHfalseEmail/password accounts, intended for dev/testing.
WAYPOINT_ALLOWED_EMAILS—Comma-separated emails allowed to sign up after the first user. Empty = closed instance.
WAYPOINT_DATA_DIRdataWhere journal photo uploads are stored (/data in the image).
WAYPOINT_TILE_URLOSM tilesRaster tile URL template ({z}/{x}/{y}).
WAYPOINT_MAP_STYLE_URL—MapLibre vector style URL; takes precedence over raster tiles. https://tiles.openfreemap.org/styles/liberty is a good free option.
WAYPOINT_NOMINATIM_URLpublic OSMGeocoding endpoint for place/station/airport search.
WAYPOINT_LANGUAGE—BCP 47 language for place names and map labels; unset keeps native names.
WAYPOINT_MCPfalseServe the MCP endpoint at /mcp.

Google sign-in

  1. In the Google Cloud Console, create a project and configure the OAuth consent screen (External; Testing mode is fine — add the accounts that will sign in as test users).
  2. Create an OAuth client ID of type Web application with the authorized redirect URI $WAYPOINT_BASE_URL/auth/google/callback.
  3. Set WAYPOINT_GOOGLE_CLIENT_ID and WAYPOINT_GOOGLE_CLIENT_SECRET and restart. The login page shows the Google button automatically.

Notes: the redirect URI must match WAYPOINT_BASE_URL exactly (a mismatch shows redirect_uri_mismatch); Google requires HTTPS except on localhost; sign in yourself first — the first account becomes admin and bypasses the allowlist. After that, only WAYPOINT_ALLOWED_EMAILS addresses can register; existing accounts always keep signing in.

Maps & geocoding

Maps default to OpenStreetMap raster tiles. For crisp vector maps with localized labels, set WAYPOINT_MAP_STYLE_URL — OpenFreeMap's liberty style is free with no key. Place search (areas, venues, stations, airports) proxies a Nominatim instance, rate-limited to respect the public server's usage policy; heavy users should self-host Nominatim and point WAYPOINT_NOMINATIM_URL at it.

Trips, areas & layers

Trips hold everything: a date range, a route, the day-by-day itinerary, journal entries, and members (owner / editor / viewer roles, added by email).

Areas are the places of the route — countries, cities, or regions, whatever scale fits the trip. They group itinerary items, order the route line on the map, and clicking one focuses the map at a zoom suited to its size.

Layers make planning collaborative. Every member can create named idea layers ("Food ideas", "Rainy day") beside the shared Main layer, each with its own color. The itinerary — on the map, the trip page, exports, even the stats — is simply the merge of all visible layers; the eye toggle includes or hides a layer everywhere. Even viewers can add items to their own layers.

Itinerary items carry a day, optional times, a category (activity, food, lodging, transport, flight, train…), and venues. Flights, trains, and transit are proper legs with departure and arrival stations — searched against real station/airport data — which the map draws as directed paths, arcing for flights.

Trip replay

Hit ▶ Replay on a trip map to watch the journey: the camera follows a dot through every point in chronological order, zooming to each leg's scale, with the vehicle emoji riding along on transport legs. A control bar offers pause, a scrubber, and speed (up to 4×). Settings can switch pacing from constant to proportional — legs then play relative to their real distance and timetable duration.

Calendar sync

The Calendar page shows every dated trip on a month grid. To get trips into your personal calendar:

  • One-off: each trip's Export section has a .ics download with the trip span and its itinerary items.
  • Live sync: Settings → Calendar sync mints a private feed URL covering all your trips. Add it to Google Calendar ("From URL") or Apple Calendar ("New Calendar Subscription") and it stays updated. The link is the only credential — reset it there if it ever leaks.

Sharing & export

Trip owners can mint read-only share links for people without accounts — they see the map, itinerary, and journal, nothing else. Every trip also exports as GPX (stops + route), GeoJSON (with journal points), Markdown (a zip with photos), ICS, and a print-friendly page for PDF.

AI access (MCP)

Set WAYPOINT_MCP=true and Waypoint serves a Model Context Protocol endpoint at /mcp, letting any MCP client — Claude, for instance — list, create, and manage your trips. The killer use case is backfilling: paste years of old travel notes into your AI of choice and let it populate Waypoint via tools like create_trip, add_area (auto-geocoded), and add_item.

Create your personal bearer token in Settings → AI access, then connect:

claude mcp add --transport http waypoint \
  https://your-host/mcp \
  --header "Authorization: Bearer YOUR_TOKEN"

Tools run with exactly your account's access — trip roles apply. The token can be rotated or disabled in Settings at any time; the endpoint is off entirely unless the server flag is set.

Releases & images

Every merge to main bumps the patch version, tags the repo, and publishes a multi-arch image to ghcr.io/ctrl-research/waypoint as X.Y.Z and latest ([minor]/[major] in the PR title bump higher). Curated GitHub Releases with notes — optionally with an extra image tag like stable — are cut from the Release workflow in the Actions tab. The running version is logged at startup.

Data & backups

All state lives in two places: the Postgres database (trips, itineraries, journal text, users, sessions) and the data directory (journal photo uploads; /data in the container). Back up both — a pg_dump plus a copy of the volume is a complete backup. Migrations run automatically on upgrade, so restoring into a newer version just works.
